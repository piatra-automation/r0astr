# Story 3.1: Safe Dependency Audit Fixes

## Status
Approved

## Story
**As a** maintainer,
**I want** all non-breaking dependency vulnerabilities resolved via `npm audit fix`,
**so that** the low-hanging security issues are eliminated before tackling the major upgrade.

## Acceptance Criteria
1. `npm audit fix` (without `--force`) applied successfully -- only non-breaking semver-compatible updates
2. The following advisories are resolved:
   - `@isaacs/brace-expansion` ReDoS (GHSA-7h2j-956f-4vf2, HIGH)
   - `axios` DoS via `__proto__` (GHSA-43fc-jf86-j433, HIGH)
   - `ajv` ReDoS (GHSA-2g4f-4pwh-qvx6, MODERATE)
   - `lodash` prototype pollution (GHSA-xxjr-mmjv-4gpg, MODERATE)
   - `qs` DoS (GHSA-w7fw-mjwx-w883, LOW)
3. `npm ci && npm run build` succeeds after lockfile update
4. `npm run dev` starts without errors
5. `package-lock.json` updated and committed -- `package.json` unchanged (these are transitive dep fixes)
6. Remaining audit output documented: only `electron-builder`-rooted and `@strudel/csound`-rooted vulnerabilities should remain

## Tasks / Subtasks
- [ ] **Task 1: Run npm audit fix** (AC: 1, 2)
  - [ ] Run `npm audit fix` (no `--force`)
  - [ ] Verify the 5 target advisories are resolved in `npm audit` output
  - [ ] If any target advisory persists, investigate and document why
- [ ] **Task 2: Verify build integrity** (AC: 3, 4)
  - [ ] Run `npm ci` to verify lockfile consistency
  - [ ] Run `npm run build` to verify production build succeeds
  - [ ] Run `npm run dev` and confirm dev server starts
  - [ ] Quick smoke test: load app, play a panel, open settings
- [ ] **Task 3: Document remaining vulnerabilities** (AC: 6)
  - [ ] Run `npm audit` and capture remaining advisories
  - [ ] Confirm remaining issues are only `electron-builder` (Story 3.2) and `@strudel/csound` (no fix available)

### Technical Constraints
- Do NOT use `npm audit fix --force` -- that would attempt the electron-builder major bump which is Story 3.2's scope
- `npm audit fix` only updates within semver-compatible ranges, so risk of breakage is minimal
- The `@strudel/csound` vulnerabilities cannot be fixed (upstream `@csound/browser` depends on `rimraf@3` which pulls vulnerable `glob`/`minimatch`) -- this is a known unfixable transitive chain
