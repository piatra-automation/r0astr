# Story 1.4: GitHub Pages Deployment (MkDocs + Lite App)

## Status
Approved

## Story
**As a** maintainer,
**I want** the documentation site and lite web app to rebuild and deploy to GitHub Pages on every push to `main`,
**so that** the public site at r0astr.org stays current with the latest docs and app version.

## Acceptance Criteria
1. Reusable workflow at `.github/workflows/deploy-pages.yml` accepting `workflow_call` (also triggerable standalone on push to `main`)
2. Builds MkDocs Material site: Python 3.x setup, `pip install mkdocs-material`, `mkdocs build --site-dir _site`
3. Builds lite app: Node.js 20 with npm cache, `npm ci`, `npm run build:lite`, copies `dist-lite/` contents into `_site/app/`
4. Deploys combined `_site/` to GitHub Pages via `actions/upload-pages-artifact` + `actions/deploy-pages@v4`
5. Uses current stable Python 3.x -- no version pinning to 3.11; installs `setuptools` via pip if needed for node-gyp compatibility
6. Caches pip dependencies (`~/.cache/pip`) and npm dependencies (`~/.npm`) for faster builds
7. Docs source is `docs-source/` directory as configured in `mkdocs.yml`
8. CNAME file for `r0astr.org` is preserved in the deployed `_site/` root

## Dev Notes

### Current State
- Existing `deploy-pages.yml` works but has fragility issues:
  - Pins Python to 3.11 for distutils compatibility
  - Runs `rm -rf node_modules package-lock.json && npm install` instead of `npm ci`
  - No dependency caching
- MkDocs config: `mkdocs.yml` at repo root, docs in `docs-source/`, output to `_site`
- Lite build config: `vite.config.lite.mjs` with base path `/app/`
- Site URL: `https://r0astr.org` (custom domain via CNAME)

### Key Files
- `.github/workflows/deploy-pages.yml` (rewrite)
- `mkdocs.yml` (existing, unchanged)
- `vite.config.lite.mjs` (existing, unchanged)
- `docs-source/` (MkDocs source files)
- `docs-source/CNAME` (if exists, or needs to be created)

### Technical Constraints
- GitHub Pages deployment requires the `github-pages` environment and OIDC token
- `actions/deploy-pages` requires `actions/configure-pages` to be called first
- The lite build aliases `websocketManager` to a lite stub (`websocketManager.lite.js`) -- this is handled by `vite.config.lite.mjs`
- `npm ci` requires committed `package-lock.json` -- verify it is tracked and up to date
- MkDocs Material may need additional plugins installed depending on `mkdocs.yml` config (check `plugins:` section)

### Site Structure After Deploy
```
r0astr.org/
  index.html          # MkDocs home
  getting-started/    # MkDocs pages
  learn/              # MkDocs pages
  app/                # Lite web app
    index.html        # Vite SPA
    assets/           # Bundled JS/CSS
  CNAME               # Custom domain
```

## Testing

- Verify `r0astr.org` loads the MkDocs site
- Verify `r0astr.org/app/` loads the lite web app
- Verify pip and npm caches are populated on second run (check cache hit in logs)
