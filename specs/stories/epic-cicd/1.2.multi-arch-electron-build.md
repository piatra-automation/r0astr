# Story 1.2: Multi-Arch Electron Build Matrix

## Status
Approved

## Story
**As a** maintainer,
**I want** a reusable build workflow that produces signed Electron artifacts for macOS (Intel + ARM), Windows x64, and Linux x64 in parallel,
**so that** releases include all platform binaries built consistently from the same source.

## Acceptance Criteria
1. Reusable workflow at `.github/workflows/build-electron.yml` accepting version string as input
2. Build matrix: `macos-latest` (dmg + zip, universal or x64 + arm64), `windows-latest` (exe, x64), `ubuntu-latest` (AppImage + deb, x64)
3. Each job: checkout, Node.js 20 setup with npm cache, `npm ci`, version injection into `package.json`, platform-specific `electron-builder` invocation
4. macOS builds use existing `build-resources/entitlements.mac.plist` and codesigning secrets from environment
5. All artifacts uploaded via `actions/upload-artifact@v4` with naming convention `electron-{platform}-{arch}`
6. Builds use `npm ci` with lockfile -- no destructive `rm -rf node_modules` hack
7. Build failures on one platform do not cancel other platform builds (`fail-fast: false`)

## Tasks / Subtasks
- [ ] **Task 1: Create build-electron.yml reusable workflow** (AC: 1, 2, 7)
  - [ ] Define `workflow_call` with `version` input
  - [ ] Define build matrix with OS, platform flags, and artifact names
  - [ ] Set `fail-fast: false` on matrix strategy
- [ ] **Task 2: Implement per-platform build steps** (AC: 3, 6)
  - [ ] Checkout with full history
  - [ ] Setup Node.js 20 with `npm` cache
  - [ ] Run `npm ci` (no clean hacks)
  - [ ] Inject version into `package.json` via `npm version --no-git-tag-version`
  - [ ] Run `electron-builder` with platform-specific flags
- [ ] **Task 3: macOS code signing** (AC: 4)
  - [ ] Import codesigning certificate from secrets
  - [ ] Set `CSC_LINK` and `CSC_KEY_PASSWORD` environment variables
  - [ ] Apply entitlements from `build-resources/entitlements.mac.plist`
  - [ ] Build for both x64 and arm64 (or universal)
- [ ] **Task 4: Artifact upload** (AC: 5)
  - [ ] Upload platform artifacts with consistent naming
  - [ ] Include only distributable files (dmg, zip, exe, AppImage, deb)
  - [ ] Exclude build intermediates

## Dev Notes

### Current State
- Existing `release.yml` has three separate jobs: `build-mac`, `build-windows`, `build-linux`
- macOS build uses `macos-13` (Intel) -- consider `macos-latest` (ARM) or `macos-14` for Apple Silicon native builds
- Current workflow deletes `node_modules` and `package-lock.json` before install -- this was a workaround for rollup optional dependency issues that `npm ci` should handle correctly
- Windows and Linux builds are straightforward

### Key Files
- `.github/workflows/build-electron.yml` (new)
- `electron-builder.json` (existing build config)
- `build-resources/entitlements.mac.plist` (existing)
- `build-resources/icons/` (existing platform icons)
- `package.json` (version field to inject)

### Technical Constraints
- `electron-builder` reads `electron-builder.json` for platform config
- macOS notarization requires Apple ID credentials if enabled (check current setup)
- `npm ci` requires `package-lock.json` to be committed and in sync with `package.json`
- GitHub-hosted macOS runners: `macos-latest` is now ARM (M1) -- for x64+arm64 cross-compilation, use `--mac --x64 --arm64` flags on electron-builder
- Artifact retention: default 90 days is fine for release assets

### electron-builder.json Reference
```json
{
  "appId": "com.piatra.r0astr",
  "productName": "r0astr",
  "directories": { "output": "release" },
  "mac": { "target": ["dmg", "zip"], "category": "public.app-category.music" },
  "win": { "target": ["nsis", "portable"] },
  "linux": { "target": ["AppImage", "deb"] }
}
```
