# Story 2.3: Panel Template and Component Refresh

## Status
Approved

## Story
**As a** user,
**I want** panels, buttons, modals, sliders, and inputs to follow the glassmorphism patterns with translucent surfaces, white-at-opacity borders, and tight spacing,
**so that** all interactive UI elements are visually cohesive with the new design language.

## Acceptance Criteria
1. **Buttons -- primary:** `var(--color-primary)` background, `var(--text-primary)` text, `border-radius: var(--radius-lg)`, `hover: opacity 0.9`, `disabled: opacity 0.5`
2. **Buttons -- ghost:** `border: 1px solid var(--border-default)`, transparent background, `var(--text-secondary)` text, `hover: var(--surface-accent)` background
3. **Modals:** overlay uses `oklch(0.04 0 0 / 0.8)` with `backdrop-filter: blur(4px)`; dialog panel uses `var(--surface-card-80)`, `var(--border-default)`, `border-radius: var(--radius-xl)`, padding 24px, `backdrop-filter: blur(12px)`
4. **Text inputs:** `var(--color-background)` background, `var(--border-default)` border, `border-radius: var(--radius-lg)`, `focus: ring 2px var(--color-primary)`, `placeholder: var(--text-muted)`
5. **Sliders:** track styled with `var(--surface-card-80)`, thumb uses `var(--color-primary)`, value display uses `var(--text-secondary)` at `--font-size-xs`
6. **Status badges:** `font-size: 11px`, `background: oklch(1 0 0 / 0.06)`, `padding: 2px 8px`, `border-radius: 9999px`, `color: var(--text-muted)`
7. **Section headers (sticky group dividers):** `font-size: 10px`, `text-transform: uppercase`, `letter-spacing: 0.05em`, `color: var(--text-muted)`, `background: oklch(1 0 0 / 0.05)`
8. **Selection lists:** `max-height` with `overflow-y: auto`, `border: 1px solid var(--border-default)`, rows with `padding: 10px 16px`, `hover: var(--surface-accent)`, `border-bottom: 1px solid var(--border-subtle)`, last row no border; selected state: `oklch(0.55 0.15 150 / 0.2)` background
9. **Panel number badge:** 24px circle, `var(--surface-card-80)` background, `var(--color-primary)` text, subtle green glow on panel hover
10. **Panel summary row:** tight padding (`var(--gap-sm)` vertical, `var(--gap-md)` horizontal), flex layout with `gap: var(--gap-sm)`
11. **Error messages:** `var(--color-error)` left border (4px), `oklch(0.6 0.2 25 / 0.1)` background, `var(--text-primary)` text
12. **Button-in-hole pattern:** replaced with flat glassmorphism buttons (the skeuomorphic bevel/hole pattern is removed entirely)
13. All component colors use oklch() or reference oklch-based CSS variables -- no hex/hsl

## Dev Notes

### Current Button Pattern (to be removed)
The existing UI uses a skeuomorphic "button-in-hole" pattern:
```css
.btn-hole { /* outer recessed container */ }
.btn-surface { /* raised inner button with bevel lighting */ }
```
This is being replaced entirely with flat glassmorphism buttons. The `.btn-hole` and `.btn-surface` classes and their CSS can be deleted. Any HTML using these classes will need the class names updated to `.btn-primary` or `.btn-ghost`.

### Key Files
- `static/css/components.css` (~400 lines -- major rewrite)
- `static/css/panels.css` (~250 lines -- moderate updates)
- `public/skins/default/templates/panel.html` (may need class name updates if button classes change)
- `public/skins/default/templates/slider.html`
- `public/skins/default/templates/slider-collapsed.html`

### Template Class Dependencies
The JavaScript expects these classes to exist (from skinManager exploration):
- `.level-panel`, `.panel-number-badge`, `.panel-title`
- `.panel-actions-left`, `.panel-actions`
- `.btn-playback`, `.btn-delete`, `.btn-duplicate`
- `.code-editor`, `.code-editor-wrapper`, `.error-message`
- `.panel-editor-container`, `.panel-controls-container`
- `.leaf-slider`, `.slider-value`

These class names MUST be preserved. New styling classes can be added, but functional classes cannot be renamed without updating JS.

### Technical Constraints
- Range input styling requires vendor-prefixed pseudo-elements: `::-webkit-slider-thumb`, `::-webkit-slider-runnable-track`, `::-moz-range-thumb`, `::-moz-range-track`
- Modal scale animation requires `transform` transition -- this is the ONE allowed exception to the "no layout animations" rule
- Button class changes in CSS must match any class references in panel templates and `index.html`
