# Story 2.2: Default Skin Theme Rewrite

## Status
Approved

## Story
**As a** user,
**I want** the default skin to render a dark forest-green glassmorphism aesthetic with translucent surfaces and green glow accents,
**so that** the app has a cohesive, dense developer-tool look out of the box.

## Acceptance Criteria
1. `public/skins/default/theme.css` fully rewritten to consume the new oklch design tokens from Story 2.1
2. Background: near-black (`--color-background`) with subtle diagonal gradient bleeding green (`oklch(0.12 0.05 150)`) into corners
3. Panel surfaces use `var(--surface-card-50)` or `var(--surface-card-80)` -- translucent, no solid fills
4. All borders use white-at-opacity tokens (`var(--border-default)`) -- never solid white, never gray
5. `backdrop-filter: blur(12px)` applied only to: top menu bar (when sticky/floating), modals -- NOT on cards or panels
6. Card/panel shadows near-zero: `box-shadow: 0 1px 3px oklch(0 0 0 / 0.05), 0 1px 2px oklch(0 0 0 / 0.03)`
7. Hoverable panels get green glow on hover: `box-shadow: 0 0 25px oklch(0.7 0.2 150 / 0.25)`
8. All transitions use `transition: color 150ms, opacity 150ms, background-color 150ms, border-color 150ms, box-shadow 150ms` -- never width, height, or layout properties
9. Modal entrance animation: `scale(0.95) -> scale(1)` over 200ms -- this is the only transform animation
10. Zero hex or hsl values in the file -- all oklch()
11. `public/skins/default/skin.json` updated with CSS variable overrides matching the green palette (if any skin-specific overrides are needed beyond the global tokens)
12. The synthwave gradient background is completely removed
13. Panel number badge: circular, uses `var(--color-primary)` tint, subtle glow on hover

## Dev Notes

### Current Default Theme (to be replaced)
- `public/skins/default/theme.css` is 508 lines
- Uses synthwave gradient: `linear-gradient(45deg, #2d1b4e, #0a0a0a, #0d3b3b)`
- Solid dark colors: `#333`, `#444`, `#555`, `#1a1a1a`
- Glow effects in cyan and purple
- Panel badges go gray -> purple when playing

### New Design Language Summary
- Background: near-black with subtle green bleed
- Surfaces: white-at-opacity layers (never opaque)
- Borders: white-at-opacity (never solid)
- Blur: earned by floating elements only
- Shadows: near-zero except green glow on hover
- Transitions: color/opacity only, 150ms
- Colors: oklch exclusively

### Key Files
- `public/skins/default/theme.css` (full rewrite)
- `public/skins/default/skin.json` (update)
- `static/css/variables.css` (consumed, from Story 2.1)

### Technical Constraints
- `backdrop-filter: blur()` has performance cost -- limit to elements that genuinely overlay content
- oklch gradient transitions may show banding on some displays -- test with subtle gradients
- The theme.css must work with the existing HTML template structure (panel.html, slider.html)
- CodeMirror editor has its own theme -- this story does NOT restyle the editor (that's Story 2.4)
- The glass skin's theme.css is separate and unaffected by this rewrite

### Color Recipes (oklch)
```css
/* Background gradient corners */
oklch(0.06 0.04 150)  /* subtle green tint */
oklch(0.04 0 0)       /* pure near-black */

/* Surface layers */
oklch(1 0 0 / 0.03)   /* card-30: barely visible */
oklch(1 0 0 / 0.05)   /* card-50: default card */
oklch(1 0 0 / 0.08)   /* card-80: elevated card */
oklch(1 0 0 / 0.1)    /* accent: hover state */

/* Green glow */
oklch(0.7 0.2 150 / 0.25)  /* hover glow */
oklch(0.55 0.15 150)       /* primary brand */
```

## Testing
- Load app with default skin -- background should be near-black with green gradient in corners
- Panels should appear translucent with green gradient bleeding through
- Hover a panel -- green glow should appear smoothly (150ms)
- Open settings modal -- backdrop should blur, modal should scale in
- Verify NO blur on panel cards
- Verify no hex values in computed styles
- Test on both Electron and browser (Vite dev server)
