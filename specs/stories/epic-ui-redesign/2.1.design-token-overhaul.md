# Story 2.1: Design Token Overhaul -- oklch Color System

## Status
Ready for Review

## Story
**As a** developer,
**I want** the CSS custom properties in `variables.css` converted to oklch color space with a forest-green (hue 150) brand palette and tightened spacing/typography tokens,
**so that** all downstream skin styling uses the new perceptually uniform color system and dense layout defaults.

## Acceptance Criteria
1. All color values in `static/css/variables.css` converted from hex/rgba to `oklch()` syntax
2. Brand primary color: oklch with hue ~150 (forest green), chroma ~0.15, lightness ~0.55 -- exposed as `--color-primary`
3. Primary glow variant: `--color-primary-glow: oklch(0.7 0.2 150 / 0.25)`
4. Background near-black: `--color-background: oklch(0.04 0 0)`
5. White-at-opacity border tokens:
   - `--border-default: oklch(1 0 0 / 0.1)` (cards, modals, inputs, dividers)
   - `--border-subtle: oklch(1 0 0 / 0.05)` (list item separators)
   - `--border-emphasis: oklch(1 0 0 / 0.2)` (dashed add-new buttons)
6. Semantic accent tokens (accent-only, not structural):
   - `--color-success: oklch(0.7 0.2 150)` (green -- active, running)
   - `--color-warning: oklch(0.75 0.15 85)` (amber -- idle, starting)
   - `--color-error: oklch(0.6 0.2 25)` (red -- error, destructive)
   - `--color-info: oklch(0.65 0.15 240)` (blue -- info banners)
   - `--color-compile: oklch(0.6 0.15 300)` (purple -- compile/context ops)
7. Surface tokens as white-at-opacity layers:
   - `--surface-card-30: oklch(1 0 0 / 0.03)`
   - `--surface-card-50: oklch(1 0 0 / 0.05)`
   - `--surface-card-80: oklch(1 0 0 / 0.08)`
   - `--surface-accent: oklch(1 0 0 / 0.1)`
8. Foreground tokens:
   - `--text-primary: oklch(0.95 0 0)` (near-white)
   - `--text-secondary: oklch(0.95 0 0 / 0.7)`
   - `--text-muted: oklch(0.95 0 0 / 0.5)`
9. Typography base size unchanged at 14px; metadata/label sizes: 10-12px available via existing `--font-size-sm` (12px) and `--font-size-xs` (10px)
10. Spacing tokens tightened: `--gap-xs: 2px`, `--gap-sm: 4px`, `--gap-md: 8px`, `--gap-lg: 12px`, `--gap-xl: 16px`
11. Existing glass skin CSS variable overrides in `glass/skin.json` remain functional -- glass skin may diverge but must not break
12. No hex or hsl values remain in `variables.css` after this story

### Technical Constraints
- oklch browser support: all modern browsers (Chrome 111+, Firefox 113+, Safari 15.4+) -- Electron 39 ships Chromium 134, so fully supported
- `color-mix()` in oklch is also supported in the same browser versions
- The `--panel-*` computed colors using `color-mix()` can remain as `color-mix(in oklch, ...)` syntax
- Some colors in `variables.css` are used by `base.css` etc. directly -- those files consume variables but should not need changes if variable names are preserved or mapped

### oklch Quick Reference
```
oklch(Lightness Chroma Hue / Alpha)
  Lightness: 0 (black) to 1 (white)
  Chroma: 0 (gray) to ~0.4 (max saturation, varies by hue)
  Hue: 0-360 degrees (150 = green, 240 = blue, 25 = red, etc.)
  Alpha: 0 (transparent) to 1 (opaque)
```
